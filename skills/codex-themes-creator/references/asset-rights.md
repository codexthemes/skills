# Asset and source rights checklist

Complete this before packaging or submission.

## Allowed

- original CSS and documentation written for the theme
- original artwork created by the author
- generated artwork whose service terms permit the intended distribution
- public-domain or permissively licensed assets with required attribution
- user-supplied assets when the user confirms redistribution rights

## Do not package without explicit rights

- reference screenshots
- private user screenshots or files
- celebrity or public-figure photographs and likenesses
- game, film, anime, sports, or brand artwork
- logos, icons, fonts, stock media, or other themes copied from third parties
- assets whose license is missing or unclear

## Required source record

Record for every asset:

- filename
- creator or source
- source URL when public
- license or permission
- required attribution
- whether commercial redistribution is allowed
- whether modification is allowed

High-level inspiration does not transfer ownership. Extract mood, palette, material, and composition principles, then create new distributable assets and CSS.
